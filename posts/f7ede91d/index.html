基础语法

  • 整型和浮点型运算时，整型会自动提升为浮点型。

  • 可以将浮点型强制转为整型，但超出范围后将始终返回整型的最大值。

  • 三元运算b ? x : y后面的类型必须相同，三元运算也是“短路运算”，只计算x或y。

  • \u#### 表示一个Unicode编码的字符。

  • Java的字符串除了是一个引用类型外，还有个重要特点，就是字符串不可变。

    • 字符串变量s再次赋值不是s内容变了，而是s指向变了。

  • 引用类型的变量可以指向一个空值null，它表示不存在，即该变量不指向任何对象。

  • 基本类型的变量是“持有”某个数值，引用类型的变量是“指向”某个对象。

  • 注意数组是引用类型，并且数组大小不可变。

  • %x格式化输出十六进制整数。

  • %e格式化输出科学计数法表示浮点数。

  • 注意，由于%表示占位符，因此，连续两个%%表示一个%字符本身。

  • Java提供的输出包括：System.out.println() / print() / printf()，其中printf()可以格式化输出。

  • Java提供Scanner对象来方便输入，读取对应的类型可以使用：scanner.nextLine() / nextInt() / nextDouble() / …

  • 整数运算在除数为0时会报错，而浮点数运算在除数为0时，不会报错，但会返回几个特殊值：

    • NaN表示Not a Number
    • Infinity表示无穷大
    • -Infinity表示负无穷大

  • 要注意浮点数判断相等不能直接用==运算符。

  • 引用类型判断内容相等要使用equals()，注意避免NullPointerException。

成员变量与局部变量的区别

  • 成员变量
    • 成员变量定义在类中，在整个类中都可以被访问。
    • 成员变量随着对象的建立而建立，存在于对象所在的堆内存。
    • 成员变量有默认初始化。
  • 局部变量
    • 局部变量值定义在局部范围内。
    • 局部变量存在于栈内存中。
    • 作用范围结束，变量会自动释放。
    • 局部变量没有默认初始化。

匿名对象

  • 例如:new Car().ran()
  • 匿名对象是对象的简化形式
  • 匿名对象两种使用情况：
  • 当对象方法只调用一次。
  • 匿名对象可以作为实际参数。

封装

  • 封装：是指隐藏对象的属性和实现细节，仅对外提供公关访问方式。
  • 好处：
    • 将变化隔离
    • 便于使用
    • 提高重用性
    • 提高安全性
  • 封装原则：
    • 将不需要对外提供的内容都隐藏起来。
    • 把属性都隐藏，提供公共方法对其访问。
  • 高内聚，低耦合
    • 高内聚就是类的内部数据操作细节自己完成，不允许外部干涉
    • 低耦合：仅暴露少量的方法给外部使用。
  • 属性私有、get/set

    pribate(私有)关键字

  • private关键字
    • 是一个权限修饰符
    • 用于修饰成员（成员变量和成员函数）
    • 被私有化的成员只存在于本类中有效
  • 常用之一
    • 将成员变量私有化，对外提供对应的set，get方法对其进程访问。提高对的数据访问的安全性。

      构造函数

  • 特点：
    • 函数名与类名相同
    • 不用定义返回值类型
    • 不用写return语句，但可以写。
  • 作用：
    • 给对象进行初始化
  • 注意：
    • 默认构造函数的特点
    • 多个构造函数以重载的形式存在
  • 什么时候定义？
  • 该食物一存在就具备一些特性或者行为，那么将这些内容定义在构造函数中。

    构造代码块

  • 作用
    • 给对象进行初始化
    • 定义不同对象共性内容
  • 特点
    • 对象一建立就运行，而且先于构造函数执行
  • 与构造函数区别
    • 构造函数给对象进行初始化
    • 构造代码块是给多有对象进行统一初始化

this关键字

  • this代表其所在函数所属对象的引用。换言之：this代本类对象的引用。
  • 什么时候使用this关键字呢？
    • 当在函数内需要用到该函数对象时，就用this
  • this()语句用于构造函数之间的调用，且只能放在第一行，因为初始化动作要在前面。

static关键字

  • static关键字：
    • 用于修饰成员（成员变量和成员函数）
  • 被修饰的对象具备以下特点：
    • 给类进行初始化
    • 随着类的加载而加载
    • 优先于对象存在
    • 被所有对象共享
    • 可以直接被类名调用,也可以被对象调用
  • 使用注意：
    • 静态方法只能访问静态成员
    • 静态方法中不可以写this,super关键字
    • 主函数是静态的
  • 静态在前加载，非静态在后。前不可访问后，后可访问前。

成员变量和静态变量的区别

  • 生命周期不同
    • 成员变量随对象的创建而存在，随着对象的被回收而释放。
    • 静态变量随着类的加载而存在，随着类的消失而消失。（生命周期较长）
  • 调用方式不同
    • 成员变量只能被对象调用。
    • 静态变量可以被对象调用，还可以被类名调用
  • 别名不同
    • 成员变量也称为实例变量
    • 静态变量称为类变量
  • 数据存储位置不同
    • 成员变量存储在对内存对象中，所以也叫对象的特有数据。
    • 静态变量存储在方法区（的静态区），所以也叫对象的共享数据。

主函数

  • public:应为权限必须最大
  • static:不需要对象，直接用主函数所属类名调用即可
  • void:主函数没有具体的返回值
  • main:函数名，不是关键字，只是一个jvm识别的固定的名字
  • String[] args:这是主函数的参数列表，是一个数组类型的参数，而且元素都是字符串类型

继承

  • 概念
    • 多个类中存在相同属性和行为时，将这些内容抽取到单独一个类中，那么多个类无需再定义这些属性和行为，只要继承单独的那个类即可。
    • 父类又称超类。
    • 子类可以直接访问父类中的非私有的属性和行为。
    • 通过 extends 关键字让类与类之间产生继承关系。
      eg: class SubDemo extends Demo{}
    • 继承的出现提高了代码的复用性。
    • 继承的出现让类与类之间产生了关系，提供了多态的前提。
  • 特点
    • Java 只支持单继承，不支持多继承。
    • Java 支持多层继承（继承体系）。
    • 定义继承需要注意：
      • 不要仅为了获取其他类中的某个功能而去继承。
      • 类与类直接按要有所属is a关系，xx1 是 xx2的一种。

super关键字

  • super 和 this 的用法相同。
  • this 代表本类应用。
  • super 代表父类引用。
  • 当子父类出现同名成员时，可以使用super 进行区分。
  • 子类要调用父类构造函数时，可以使用super语句。

函数覆盖（Override）

  • 子类中出现与父类一模一样的方法时，会出现覆盖操作，也成为重写或复写。
  • 父类中的私有方法不可以被复写。
  • 在子类覆盖方法中，继续使用被覆盖的方法可以通过 super.fun_name获取。
  • 方法名相同，参数列表相同
  • 修饰符：范围可以扩大但不可以缩小 public > protected > default > private
  • 覆盖注意事项：
    • 覆盖时，子类方法权限一定要大于父类方法权限。
    • 静态只能覆盖静态。
  • 覆盖的应用：当子类需要父类的功能，而功能主体子类有自己的特有内容时，可以复写父类中的方法，这样，即沿袭了父类的功能，有定义了子类特有的内容。

子类实例化过程

  • 子类中所有的构造函数默认都会访问父类中空参数的构造函数。
  • 因为每一个构造函数的第一行都有以条默认的语句 super(); 显化写出来一定要写在子类无参构造第一行
  • 如果父类只有有参构，子类不可以写无参构造（除非里面调用父类有参构造），所以在所有类的书写时，写有参构造之前先写无参构造。
  • 子类会具备父类中的数据，所以要先明确父类是如何对这些数据初始化的。
  • 当父类中没有空参数的构造函数时，子类的构造函数必须通过this或者super语句指定要访问的构造函数。

final关键字

  • final可以修饰类/方法/变量。
  • final修饰的类不可以被继承。
  • final修饰的方法不可以被覆盖。
  • final修饰的变量是一个常量。只能被赋值一次。
  • 内部类只能访问被final修饰的局部变量。

抽象类

  • 抽象就是从多个事物中将共性的/本质的内容抽取出来。
  • Java 中可以定义没有方法体的方法，该方法的具体实现由子类完成，该方法称为抽象方法，包含抽象方法的类是抽象类。
  • 多个对象都具备相同的功能，但是功能具体内容有所不同，那么在抽取过程中，只抽取了功能的定义，并未抽取功能主体，那么只有功能声明，没有功能主体的方法称为抽象方法。

抽象类的特点

  • 格式： 修饰符 abstract 返回值类型 函数名（参数列表）
  • 抽象类不可以被实例化，也就是不可以用new创建对象。
  • 抽象类通过其子类实例化，儿子类需要覆盖掉抽象类中的所有的抽象方法后才可以创建对象，否则子类也是抽象类。
  • 抽象类中可以有构造函数吗？
    • 可以。抽象类的构造函数用来初始化抽象类的一些字段，而这一切都在抽象类的派生类实例化之前发生。不仅如此，抽象类的构造函数还有一种巧妙的应用： 就是在其内部实现子类必须执行代码。
  • 抽象关键字abstract不可以和哪些关键字共存？
    • private: 因为一个abstract方法需要被重写，所以不能修饰为pivate
    • final:因为一个abstract方法需要被重写，被finall修饰的方法是不能被重写的。
    • static:因为一个abstract方法没有方法体，静态方法需要对方法体执行内容分配的空间（abstract是没有实现的，不能产生对象，而是static是属于类的，类本身是已经存在的对象）
    • sysnchronized:是同步的，然而同步的需要具体的操作才能同步，但abstract是只有声明没有实现（即：使用sysnchronized关键字的是需要有具体的实现同步操作的，但是使用 abstract 是只有声明而没有实现的，这样就产生了冲突）
    • native: 他们本身的定义就是冲突的,native声明的方法是移交本地操作系统实现的，而 abstract 是移动子类对象实现的，同时修饰的话，导致不知道谁实现声明的方法
  • 抽象类中不可以没有抽象方法？
    • 抽象类可以没有抽象方法，但是如果你的一个类已经声明成了抽象类，即使这个类中没有抽方法，它也不能直接构造一个该类的对象。但是含抽象方法的类是抽象类。

接口

  • 格式:interface{}
  • 接口中的成员修饰符是固定的。
    • 成员常量：public static final
    • 成员函数： public abstract
  • 接口的出现将“多继承”通过另一种形式体现出来，即“多实现”。
  • 接口的特点：
    • 接口是对外暴漏的规则。
    • 接口是程序的功能拓展。
    • 接口可以用来多实现。
    • 类与接口之间是实现关系，关键字implement ,而且类可以继承一个类的同时实现多个接口。(实现类命名规则:接口nameImpl)
      eg: class Test extends Demo implements InterA,InterB,...
  • 接口与接口之间可以有继承关系。(而且有多继承)

多态

  • 成员函数(非静态【有重写特性】)：
    • 编译时：要看引用变量所属的类中是否有所调用的成员。
    • 运行时：要看对象所属的类中是否有所调用的成员。
      • 一句话：成员函数在多态调用时，编译看左边，运行看右边
      • 看声明时等号左边声明类有没有此方法（没有会报错），运行时看等号左边new哪个类，调用的就是哪个类的方法。
  • 成员函数（静态):
    • 无论编译和运行，都参考做左边。
  • 成员变量：
    • 只看引用变量所属的类。(无论编译和运行，都参考做左边)
  • instanceof用于判断对象的类型。对象 instanceof 类型（类类型 接口类型）
